# Дизайн — Aegis (ПЛАН)

> **Статус: реализуется.** Это спецификация визуального языка приложения — токены и правила; реализуется в `src/Aegis.App` (Avalonia, тёмная тема — в `App.axaml` + ресурсах). Утверждённые макеты — в [`design/mockups/`](design/mockups/).
> Стиль: **мягкий тёмный дашборд с аккуратным градиентом-акцентом**. Цель — современный, спокойный, читаемый интерфейс системного оптимизатора. UI приложения — **русский** (для себя, мультиязычие не нужно).
>
> ⚠️ Веб дизайн-система Halo (светлая, для веба) здесь **НЕ применяется** — это десктоп Avalonia. Токены ниже описаны прямо под Avalonia/XAML.

---

## 1. Принципы

- **Тёмный, но мягкий.** Глубокий графит вместо чёрного; поверхности-карточки чуть светлее фона, без резких границ.
- **Акцент — градиент, приглушённый.** Сине-фиолетовый → циан (индиго→cyan). Не кислотный, не кричащий. Используется точечно: активная вкладка, основная кнопка, прогресс, фокус.
- **Семантика статусов читаема с первого взгляда.** ok / warn / danger у каждого найденного пункта.
- **Спокойствие = доверие.** Приложение меняет системные настройки — интерфейс должен внушать уверенность, а не тревогу. Минимум визуального шума, щедрые отступы, мягкие тени.
- **Доступность не опция.** Высокий контраст текста, видимый фокус, крупные кликабельные зоны.

## 2. Цветовые токены (тёмная тема)

Имена — в стиле XAML `ResourceDictionary` (Avalonia) (ключи `x:Key`). Значения — HEX, под `SolidColorBrush` / `LinearGradientBrush`.

### Фон и поверхности
| Токен | HEX | Назначение |
|-------|-----|------------|
| `Bg.Base` | `#14161B` | Основной фон окна (глубокий графит) |
| `Bg.Sunken` | `#101216` | Утопленные зоны (лог, пустые состояния) |
| `Surface.1` | `#1B1E25` | Карточка / панель (база поверхности) |
| `Surface.2` | `#22262F` | Приподнятая поверхность (hover карточки, поповеры, тултип-фон) |
| `Surface.3` | `#2A2F3A` | Самый верхний слой (диалоги, меню, активные строки) |
| `Border.Subtle` | `#2A2E37` | Тонкая граница/разделитель |
| `Border.Strong` | `#3A4150` | Заметная граница (поля ввода, выделенная карточка) |

### Текст
| Токен | HEX | Назначение |
|-------|-----|------------|
| `Text.Primary` | `#EEF1F6` | Основной текст (контраст ≈ 14:1 на `Bg.Base`) |
| `Text.Secondary` | `#AEB6C4` | Вторичный текст, подписи |
| `Text.Muted` | `#7B8494` | Приглушённый (хинты, дизейбл-подписи, метаданные) |
| `Text.OnAccent` | `#0B0E13` | Текст поверх градиента-акцента (тёмный, для контраста на ярком) |

### Акцент (градиент)
| Токен | Значение | Назначение |
|-------|----------|------------|
| `Accent.Start` | `#6D7CFF` (индиго) | Левая точка градиента |
| `Accent.End` | `#3FD2E0` (циан) | Правая точка градиента |
| `Accent.Gradient` | `LinearGradientBrush` 135°: `Accent.Start`→`Accent.End` | Основная кнопка, активная вкладка, заполнение прогресса |
| `Accent.Solid` | `#6E86FF` | Сплошной акцент для мелких элементов (фокус-кольцо, чекбокс, ссылки) |
| `Accent.Glow` | `#6D7CFF` @ 28% opacity | Мягкое свечение под акцентными элементами |
| `Accent.Subtle` | `#6D7CFF` @ 14% opacity | Заливка-подложка активной вкладки/строки |

### Семантика статусов (пункты сканов)
| Токен | HEX | Подложка (12% opacity) | Назначение |
|-------|-----|------------------------|------------|
| `Status.Ok` | `#3FCF8E` (мягкий зелёный) | `#3FCF8E` @ 12% | Всё в порядке / исправлено |
| `Status.Warn` | `#F2B855` (янтарный) | `#F2B855` @ 12% | Предупреждение / некритично |
| `Status.Danger` | `#F2696B` (приглушённый красный) | `#F2696B` @ 12% | Проблема / угроза / требует внимания |
| `Status.Info` | `#5FB4F2` (спокойный голубой) | `#5FB4F2` @ 12% | Информация / нейтральная находка |

> Цвета статусов **приглушённые** (не «светофорные» чистые RGB) — чтобы на тёмном фоне не резали глаз и сочетались с акцентом.

## 3. Типографика

- **Семейство:** `Segoe UI Variable` (системный, нативен для Win 10/11). Запасной — `Inter`, далее `Segoe UI`.
- **Чистый sans, без декора.** Цифры — табличные (для таблиц находок и размеров мусора).

| Токен | Размер / начертание | Назначение |
|-------|---------------------|------------|
| `Type.Display` | 28 / SemiBold | Заголовок экрана / название раздела |
| `Type.Title` | 20 / SemiBold | Заголовок карточки, имя вкладки-группы |
| `Type.Subtitle` | 16 / Medium | Подзаголовок, имя пункта находки |
| `Type.Body` | 14 / Regular | Основной текст, описания, тултипы |
| `Type.Caption` | 12 / Regular | Метаданные, подписи, хинты |
| `Type.Mono` | 13 / Consolas | Пути, ключи реестра, хэши, лог |

- **Межстрочный:** ~1.45 для body. **Контраст:** body не ниже `Text.Secondary` на `Surface.1`.

## 4. Layout (раскладка приложения)

```
┌──────────────────────────────────────────────────────────┐
│  Заголовок окна: Aegis · [⚙ права администратора]          │
├────────────┬─────────────────────────────────────────────┤
│            │  Вкладки по группам сканов (верх области)     │
│  Левое     │  [Система][Драйверы][Реестр][Автозапуск]…     │
│  меню /    ├─────────────────────────────────────────────┤
│  разделы   │  Прогресс скана (видный, с градиентом)        │
│            │  ───────────────────────────────────          │
│  • Дашборд │  Список находок (Finding):                     │
│  • Сканы   │   ☑  [danger] Имя пункта        (?)  [Чинить] │
│  • Бэкапы  │   ☐  [warn]   Имя пункта        (?)  [Чинить] │
│  • Логи    │   ☑  [ok]     Имя пункта (исправлено)         │
│  • О прогр.│  ───────────────────────────────────          │
│            │  [Выделить всё]            [Исправить выбранное]│
└────────────┴─────────────────────────────────────────────┘
```

- **Левое меню (рельс):** разделы приложения — Дашборд, Сканы, **Бэкапы**, Логи, О программе. Тёмная панель `Surface.1`, активный пункт — `Accent.Subtle` + полоска `Accent.Gradient` слева.
- **Вкладки по группам сканов** (верх рабочей области): Система · Драйверы · Реестр · Автозапуск · Процессы · Системные настройки · Мусор · Угрозы · Драйверы GPU. Активная вкладка — текст `Text.Primary` + подчёркивание `Accent.Gradient`; неактивная — `Text.Secondary`.
- **Прогресс скана:** заметная полоса/кольцо с заливкой `Accent.Gradient` и мягким `Accent.Glow`; рядом — что сканируется сейчас и счётчик найденного.
- **Список находок (Finding):** строки/карточки на `Surface.1`, hover → `Surface.2`. Каждая строка: **чекбокс** (массовое выделение) · **бейдж статуса** (ok/warn/danger из семантики) · имя пункта · иконка **«?»** · кнопка «Чинить». Исправленные — приглушаются, бейдж → `Status.Ok`.
- **Раздел «Бэкапы»:** список точек восстановления / бэкап-файлов (дата, что затронуто, размер) + кнопка «Восстановить» с явным предупреждением.

## 5. Компоненты и состояния

### Иконка «?» (объяснение пункта)
- Круг 18×18, `Surface.2`, символ «?» цветом `Text.Secondary`.
- **Hover/фокус → тултип** (`Surface.2` фон, `Border.Subtle`, мягкая тень): короткий текст **простыми словами** — в чём суть проблемы и что даст её исправление и зачем. Ширина тултипа ограничена (~320px), `Type.Body`.

### Чекбоксы (массовое выделение)
- Невыбранный — рамка `Border.Strong` на прозрачном. Выбранный — заливка `Accent.Solid`, галочка `Text.OnAccent`. Крупная кликабельная зона (вся строка кликабельна по желанию).
- «Выделить всё» — заметная вторичная кнопка над/под списком.

### Кнопки
| Вид | Стиль |
|-----|-------|
| Primary | заливка `Accent.Gradient`, текст `Text.OnAccent`, `Accent.Glow` при hover |
| Secondary | `Surface.2`, текст `Text.Primary`, граница `Border.Strong` |
| Danger | контур/заливка `Status.Danger` — для необратимо выглядящих действий (с подтверждением) |
| Ghost | прозрачная, текст `Text.Secondary`, hover → `Surface.2` |

### Диалог подтверждения после ребута
- Модальное окно на `Surface.3`, затемнение фона. Заголовок «Всё работает корректно?», пояснение `Type.Body`.
- Две кнопки: **«Да, всё работает»** (Primary, оставляет изменения) и **«Нет, откатиться»** (Danger/Secondary, грузит бэкап). Кнопки крупные, разнесены.

### Бейджи статусов
- Пилюля: подложка статуса (12% opacity) + текст цветом статуса. Краткая метка: «Проблема» / «Внимание» / «OK» / «Инфо».

## 6. Тени, скругления, движение

- **Скругления:** карточки/кнопки 10px, тултипы/диалоги 14px, чекбоксы 5px.
- **Тени:** мягкие, тёмные (`#000` @ 35–45%, большой blur, малый сдвиг) — намёк на парение, без жёстких краёв. Акцентные элементы — дополнительный цветной `Accent.Glow`.
- **Движение:** короткие spring-переходы (hover, раскрытие тултипа, появление диалога) ~150–220ms; ничего «прыгающего». Прогресс скана — плавная анимация заливки.

---

## 7. Доступность

- **Контраст:** основной текст ≥ 7:1 на фоне; вторичный ≥ 4.5:1; статусные цвета подобраны так, чтобы метка читалась и без цвета (бейдж + текст).
- **Фокус:** видимое кольцо `Accent.Solid` (2px) на всех интерактивных элементах; навигация с клавиатуры по списку находок и вкладкам.
- **Размер целей:** кликабельные зоны ≥ 32px; чекбоксы и «?» — с увеличенной hit-area.
- **Не только цветом:** статус всегда дублируется текстовой меткой/иконкой (для людей с нарушением цветовосприятия).
