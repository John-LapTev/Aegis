# Архитектура — Aegis

> **Статус: в разработке** (каркас Фазы 1 создан; описана целевая архитектура). Решения фиксируются в [`decisions/`](decisions/), реальная структура — в [`PROJECT_STRUCTURE.md`](PROJECT_STRUCTURE.md). Здесь — актуальная картина задуманного.

## Стек
- **Платформа:** Windows 10/11, десктоп-приложение (самодостаточный `.exe`).
- **Язык/рантайм:** C# 13 / **.NET 9**.
- **UI:** **Avalonia** (XAML, ADR 0006), кастомная **тёмная тема** с градиент-акцентом (см. `DESIGN.md` / `DESIGN_BRIEF.md`). Собирается в Windows `.exe` на любой ОС. UI — **русский** (мультиязычие не нужно).
- **Паттерн:** **MVVM** (CommunityToolkit.Mvvm — `ObservableObject`, `RelayCommand`).
- **DI:** `Microsoft.Extensions.DependencyInjection` (+ `Hosting`/`Options`/`Configuration`).
- **Асинхронность:** `async`/`await` повсюду; UI-поток не блокируется длинными операциями (сканы/правки — в фоне).
- **Доступ к системе:** WMI (`System.Management`), реестр (`Microsoft.Win32.Registry`), службы/автозапуск, Process API; при необходимости — P/Invoke в **изолированных безопасных обёртках**.
- **Логирование:** **Serilog** (файл + структурные логи критичных операций).
- **Тесты:** **xUnit** (логика сканеров и бэкапа).
- **Публикация:** single-file, self-contained, **win-x64**.
- **Права:** `app.manifest` с `requireAdministrator` (элевейшн при старте).

## Обзор
Высокоуровневая схема: **оболочка-приложение** запускает **движок сканов** (набор плагинов `IScanner`), результаты (`Finding`) раскладываются по **вкладкам** групп. Исправления идут через **FixService** поверх **BackupService** (обратимость). Перезагрузки координирует **RebootFlow**. Угрозы — **ThreatScanner**, GPU-тюнинг — **GpuTuner**. Всё связано через DI и логируется Serilog.

## Модули и компоненты (ПЛАН)

### 1. Оболочка приложения (`Aegis.App`, Avalonia/MVVM)
- Главное окно с **вкладками по группам сканов** — поэтапный проход по найденному.
- Каждый пункт (`Finding`) — со значком **«?» в круге**: при наведении короткий текст простыми словами (поле `Explain`) — суть проблемы и что даст её исправление.
- **Чекбоксы** для массового выделения: «выделить всё» → исправить разом, либо чинить **по одному**.
- Видимый **прогресс** скана; раздел **«Бэкапы»** (выбрать точку/файл и восстановить); **диалог после ребута** («Всё работает корректно? Да / Нет, откатиться»).
- Доступность: высокий контраст текста на тёмном, видимый фокус, крупные кликабельные зоны.

### 2. Движок сканов (`Aegis.Core`)
- Контракт **`IScanner`**: асинхронный `ScanAsync` → коллекция **`Finding`**.
- **`Finding`** — единица результата:
  - `Severity` — `Ok` / `Warn` / `Danger` (семантика статусов для UI: зелёный / янтарный / красный).
  - `Explain` — текст для «?» простыми словами (что не так и что даст исправление).
  - `Fix` — описание/делегат обратимого исправления (передаётся в `FixService`).
- Оркестратор запускает зарегистрированные через DI сканеры, агрегирует находки по группам-вкладкам, репортит прогресс.

### 3. Сканеры-плагины (`Aegis.Scanners`, реализации `IScanner`)
Десять групп (= вкладки), каждая — отдельный сканер:
1. **System** — общий скан системы/компьютера (Win 10/11), системная информация.
2. **Junk** — поиск мусора по всей системе (очистка).
3. **Registry** — проблемы и подозрительные записи реестра.
4. **Autostart** — подозрительные элементы автозапуска.
5. **Processes** — подозрительные/вредные активные процессы.
6. **Settings** — нарушения/неправильные системные настройки.
7. **Drivers** — проверка новых версий драйверов.
8. **GPU** — настройки драйверов NVIDIA/AMD (анализ под мощность ПК).
9. **Threats** — вирусы/трояны/майнеры (делегирует в `Aegis.Threats`).
10. **Missing** — недостающие библиотеки/драйверы под модель ПК.

### 4. FixService (обратимые правки)
- Единая точка исправлений: применяет `Finding.Fix` **только после** создания страховки через `BackupService`.
- **Каждое исправление обратимо**, ничего не удаляется безвозвратно без бэкапа; явные предупреждения; полный лог (Serilog).
- Поддержка одиночного и пакетного применения (для «выделить всё»), с per-item обработкой ошибок (ошибку конкретного шага не теряем).

### 5. BackupService (System Restore + экспорт веток)
- Перед любой правкой: создаёт **точку восстановления Windows (System Restore)** и **экспортирует затрагиваемые ветки реестра/настройки** ПЕРЕД изменением.
- Хранит метаданные бэкапов для раздела «Бэкапы»: выбрать точку/экспорт и восстановить.
- Откат при **незагрузке** — через **штатное восстановление Windows (WinRE / точка восстановления)**, НЕ кастомный загрузчик.

### 6. RebootFlow (RunOnce → диалог)
- Если правка требует перезагрузки — регистрирует **авто-старт после ребута** (RunOnce / Task Scheduler).
- После ребута приложение само поднимается с **диалогом**: «Всё работает корректно?» → **Да** (изменения остаются) / **Нет, откатиться** (грузим бэкап через `BackupService`).

### 7. ThreatScanner (`Aegis.Threats`)
- **Свой эвристический сканер**: анализ процессов / автозапуска / реестра / сетевых соединений.
- **Онлайн-сверка хэшей через VirusTotal API**; опционально базовая сверка с Windows Defender (`MpCmdRun` / AMSI).
- Детект **майнеров** — эвристика по нагрузке GPU/CPU + известные паттерны.
- Ограничение зафиксировано честно: **полноценный AV-движок с сигнатурами с нуля НЕ строится** (см. ADR 0003).

### 8. GpuTuner (NVIDIA / AMD)
- Анализ и предложение оптимальных настроек драйвера под мощность ПК.
- NVIDIA — NVAPI / `nvidia-smi`; AMD — ADLX / ADL или вендор-реестр (best-effort, **поздняя фаза**).

### 9. Низкоуровневый доступ (`Aegis.System`)
- Изолированные безопасные обёртки: **WMI** (`System.Management`), **реестр** (`Microsoft.Win32.Registry`), **службы**, **Process API**, при необходимости **P/Invoke**.
- Вся работа с реестром/службами — параметризованно и с обработкой ошибок.

### Сквозные аспекты
- **Элевейшн:** `app.manifest` с `requireAdministrator` + проверка прав в коде.
- **DI:** регистрация сканеров и сервисов через `Microsoft.Extensions.DependencyInjection`.
- **Логирование:** Serilog для критичных операций (сканы, правки, бэкап, откат, обращения к VirusTotal).
- **Безопасность изменений (ключевой принцип):** любое системное изменение — через обратимую операцию (точка восстановления / бэкап ветки **ПЕРЕД** правкой).
- **Секреты:** VirusTotal API key и токен бота — через env / `.personal/`, **не в репозиторий**.

## Открытые вопросы (ПЛАН)
- [ ] GPU-тюнинг AMD: какой путь надёжнее — ADLX, ADL или вендор-реестр (best-effort).
- [ ] VirusTotal: лимиты бесплатного API и стратегия кеширования/батчинга хэшей.
- [ ] Глубина опциональной интеграции с Defender (`MpCmdRun` vs AMSI).
- [ ] Критерии эвристики для майнеров (пороги нагрузки, чтобы не плодить ложные срабатывания).
- [ ] Источник «эталонной» базы драйверов/библиотек под модель ПК (для Missing-сканера).
- [ ] Опциональный установщик в Фазе 7 vs только portable `.exe`.

## Принципы
- Простота снаружи, сложность спрятана. Очевидность. Надёжность.
- **Обратимость прежде всего:** бэкап перед каждой правкой, без безвозвратных удалений.
- Деликатность к системе: явные предупреждения, прозрачные «?»-объяснения, полный лог.
