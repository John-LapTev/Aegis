# Обнаружение и удаление майнеров — Aegis

> Ключевое требование заказчика (Иван, 2026-06-17): не просто находить **скрытые майнеры с автозапуском**, а **удалять** их — а если удалить нельзя, то ХОТЯ БЫ отключить автозапуск и довести удаление до конца после перезагрузки.
> Всё обратимо: точка восстановления + карантин ПЕРЕД действиями (ADR 0002, 0004). Никаких безвозвратных удалений.

## 1. Обнаружение (корреляция нескольких признаков)

Майнер редко выдаёт себя одним признаком — собираем «улики» из разных сканеров и повышаем уверенность при совпадении:

- **Нагрузка** — устойчиво высокая загрузка CPU/GPU, особенно в простое (idle-mining) → `ProcessesScanner` (+ GPU best-effort позже).
- **Автозапуск** — процесс прописан в автозапуске (реестр Run, папка, задача, служба) → `AutostartScanner`.
- **Сеть** — подключения к майнинг-пулам (порты Stratum 3333/4444/5555/7777/9999/14444, домены пулов) → `NetworkThreatScanner` (см. [`threat-backlog.md`](threat-backlog.md) #20/#21).
- **Маскировка** — запуск через LOLBin (powershell -enc и т.п.) или неподписанный файл из временной папки → `AutostartScanner`/`ProcessesScanner`.
- **Репутация** — сверка хэша файла через VirusTotal (ADR 0003).

Находка майнера связывает все улики в один объект (файл + процессы + автозапуск-записи + соединения), чтобы удалять комплексно.

## 2. Алгоритм удаления (многошаговый, обратимый)

```
0. Бэкап: точка восстановления + экспорт затрагиваемых веток автозапуска. Цель-файлы → копия в карантин.
1. Остановить ВСЕ процессы майнера (по пути файла и дереву процессов). НЕ трогаем критические системные процессы.
2. Снять автозапуск: удалить записи Run / отключить задачу планировщика / остановить и отключить службу.
3. Удалить файлы → перенести в КАРАНТИН (не безвозвратно; можно восстановить).
4. Проверка: процессов нет, автозапуска нет, файлы в карантине → успех.
```

### Если файл «заперт» (используется и не удаляется) ИЛИ связан с важным системным процессом

```
A. НЕ убиваем критический системный процесс (риск «положить» систему).
B. Отключаем автозапуск майнера (чтобы он не вернулся после перезагрузки).
C. Помечаем файл на удаление при следующей загрузке Windows
   (MoveFileEx MOVEFILE_DELAY_UNTIL_REBOOT / PendingFileRenameOperations).
D. Просим/планируем перезагрузку. После ребута процессов майнера уже нет →
   программа авто-стартует (RunOnce/Task Scheduler) и завершает удаление (или ОС удаляет файл сама).
E. Подтверждение «всё ок? Да / Откатиться» в рамках reboot-flow (Фаза 3).
```

## 3. Безопасность

- **Список критических процессов/служб** — защищён от остановки/удаления (чтобы не сломать загрузку Windows).
- **Карантин вместо удаления** — файл хранится зашифрованным/перемещённым N дней, восстановим из раздела «Бэкапы».
- **Бэкап перед каждым шагом** — точка восстановления + экспорт веток автозапуска.
- **Понятные тексты** (UX-правило): «нашли скрытый майнер — он грузит компьютер и работает на чужих; уберём его. Если не получится сразу — выключим запуск и дочистим после перезагрузки».

## 4. Реализация (по фазам)

- **Детект** — улики уже частично есть (`ProcessesScanner`, `AutostartScanner`, `NetworkThreatScanner`); корреляция «майнер» — отдельная находка (Фаза 4).
- **Удаление** — `IFix` (например, `MinerRemovalFix`) поверх обёрток процессов/служб/реестра/файлов (`Aegis.System`) + `IRestorePointService` + карантин (Фаза 3–4).
- **Delete-on-reboot + авто-старт после ребута** — общий механизм reboot-flow (Фаза 3).
- **Тестируемое ядро решения** — `MinerRemovalPlanner` (чистая логика): по флагам (`canDeleteNow`, `fileLocked`, `tiedToCriticalProcess`) строит упорядоченный план шагов (включая «отложить удаление на ребут»). Пишется с тестами независимо от Windows.
