## ⭐ ТОП-5 — внедрить первыми

Критерий: **высокая severity × низкая цена × частота на домашних ПК**.

1. **#36 — LOLBin в автозапуске.** Данные (`AutostartEntry.Command`) уже в пробнике — чистая логика разбора аргументов. Закрывает целый класс маскировки под подписанные утилиты MS. Максимальный ROI.
2. **#12+#13 — системный прокси WinINET + PAC/AutoConfigURL.** «Главной дыры по линзе прокси сейчас нет вообще». Один реестр-слой `Internet Settings` закрывает оба + даёт фундамент для #15/#19. Любимый приём адваре/банк-троянов.
3. **#1 — hosts-редирект на внешний публичный IP.** Чистая проверка IP поверх существующего `HostsEntry.MappedIp`, без сети. Чинит фундаментальную дыру текущей логики (всё вне короткого списка банков = «безобидно»).
4. **#20+#21 — майнинг-пулы (порты + домены).** Списки в `NetworkHeuristics` отсутствуют, но TCP-таблица и резолв уже есть. Дёшево, очень частая зараза, легко объяснить пользователю. `SuspiciousConnection.Reason` уже предусмотрен.
5. **#24+#26 — листенеры RAT + маскировка системных имён.** Сканер видит только исходящие — слепота к бэкдорам. Один слой LISTEN-таблицы (`GetExtendedTcpTable`) + обогащение `SuspiciousConnection` полями `ExecutablePath`/`Signature` закрывает сразу #24, #26, и подпирает #15/#23/#29.

**Логика порядка:** #1/#36/#37/#41 — чистая логика поверх готовых проб (внедряются за часы). #12/#13/#24/#20-21 — по одному новому слою данных каждый, но с максимальным охватом смежных индикаторов. Дорогие временные/сетевые слои (#16/#22/#25) — в последнюю фазу: высокий риск ложных срабатываний и наблюдение во времени, а не мгновенный снимок.
