# Проверка угроз — «воронка» (heuristics → Defender → VirusTotal)

> Согласовано с Иваном (2026-06-17). Расширяет [ADR 0003](../decisions/0003-threat-scanning.md).
> Цель: не полагаться на один антивирус, работать офлайн, и не упираться в лимит VirusTotal — за счёт
> поэтапного отсева, где до онлайн-проверки доходит минимум файлов.

## Этапы (каждый сокращает объём для следующего)

1. **Наши эвристики** — мгновенно, офлайн, бесплатно. Из ВСЕХ объектов отбирают только подозрительные:
   нет валидной подписи (WinVerifyTrust — включая каталоговые подписи Windows), странный путь (temp),
   автозапуск, подозрительная сеть (майнинг-пулы/Tor/hosts/DNS), маскировка под LOLBin. Сотни → единицы.
2. **Защитник Windows (Defender)** — локальная проверка реальным AV (`MpCmdRun.exe -Scan -ScanType 3 -File`),
   офлайн, без лимитов. Проверяет только подозрительные из шага 1. **Опционально** (вкл/выкл в настройках —
   доверие к Defender у пользователя ниже; используем как дополнительный сигнал, не как финал).
3. **VirusTotal** — онлайн, 70+ движков (консенсус, а не один AV). Только для немногих спорных из шага 2
   или по кнопке «Проверить онлайн». Отправляется **только SHA-256-хэш** (не файл) — приватно. Лимит
   бесплатного тарифа (~4/мин, 500/день) не мешает, т.к. сюда доходят единицы; кэшируем результаты.

## Принципы
- **Ни один AV — не истина.** Показываем все сигналы (поведение + локальный Defender + консенсус VirusTotal)
  и понятное объяснение; финальное решение — за пользователем. Без «100% вирус» на пустом месте.
- **Офлайн работает:** шаги 1–2 не требуют интернета; без сети шаг 3 помечается «не удалось проверить онлайн».
- **Кейгены/пиратское ПО** часто помечаются эвристиками/AV, но не всегда вредны — не гоним их в VirusTotal
  зря, оставляем выбор пользователю.
- **Свой полноценный AV-движок с нуля НЕ делаем** (ADR 0003): нереалистично без команды аналитиков и
  постоянной базы сигнатур. Роль «локального AV» выполняет уже установленный Defender.
