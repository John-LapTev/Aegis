## Фаза 2 — Базовые сканы + UI находок _(в основном готово)_
- [x] Логика сканеров + тесты (**23 активных** сканеров по 9 вкладкам: система, обслуживание (SFC/DISM+сброс сети), диск-SMART, температуры, мусор, очистка обновлений, большие/дубли, анализ места, остатки удалённых программ, автозапуск, процессы, реестр, настройки, приватность/деблоат, UWP-хлам, сетевые угрозы, драйверы, звук, фирменные утилиты под модель ПК).
- [x] **Windows-пробники** (`Aegis.System`) для всех реализованных групп — реальное чтение системы (реестр/файлы/процессы/диски/SMART/сеть); DI-связка. Драйверы/GPU/недостающее — отдельная фаза.
- [x] Результаты разложены по вкладкам, список `Finding` с «?»-тултипом. Авто-скан при запуске сознательно убран — пользователь сам жмёт «Проверить всё» или скан по разделу (см. CHANGELOG).
- [x] Починка по одному пункту / «выделить всё» → исправить разом. Работает: **приватность/настройки** (реклама/телеметрия/защита), **мусор** (Очистить → Корзина), **автозапуск** (Отключить), **процессы** (Остановить), **службы Xbox** (Отключить), **реестр-находки** (удаление осиротевших с экспортом ветки), **задачи планировщика** (Отключить, обратимо). Кнопки подписаны по смыслу.
- [~] Каждый `Fix` создаёт точку восстановления + бэкап значения ПЕРЕД правкой (`RestorePointService` + `RegistryBackupStore`) — обратимость для реестровых правок реализована.
- [~] Видимый прогресс скана (пока статус-текст; кольцо/полоса — позже).
- [ ] **Два режима запуска скана:** одна кнопка «Проверить всё» (все группы разом → раскладка по вкладкам) и отдельный скан по конкретной вкладке (например, только «Драйверы»). Движок уже поддерживает оба (`ScanOrchestrator.ScanAllAsync` vs одиночный `IScanner`).

## Фаза 3 — Бэкап/откат целиком _(ПЛАН)_
- [ ] Точки восстановления Windows (System Restore) + экспорт/импорт затронутых веток реестра/настроек.
- [ ] Flow перезагрузки: RunOnce / Task Scheduler → авто-старт после ребута → диалог «всё ок? Да / Откатиться» («Да» — оставить изменения, «Нет» — загрузить бэкап).
- [x] Раздел «Бэкапы» в приложении: список реестровых бэкапов + карантина с кнопкой «Откатить» (`RestorePointService.ListBackupsAsync`/`RestoreAsync`).
- [ ] Откат при незагрузке — через штатное восстановление Windows (WinRE / точка восстановления), без кастомного загрузчика.

## Фаза 4 — Сканер угроз _(начато)_
- [~] Эвристика: процессы, автозапуск, реестр, сетевые соединения (`NetworkThreatScanner`, `ProcessesScanner`, `AutostartScanner` + LOLBin — логика готова; ещё индикаторы — в [`security/threat-backlog.md`](security/threat-backlog.md)).
- [x] Онлайн-сверка хэшей через VirusTotal API + локальный Защитник Windows (воронка `FileReputationCheck`); кнопка «Проверить онлайн» у подозрительных файлов. Троттлинг + кеш по хэшу (`ThrottledCachingReputationService`): лимит ≈4/мин соблюдается, при исчерпании — честное «VT пропущен, проверено Защитником».
- [ ] **Детект и удаление майнеров** — корреляция признаков (CPU/автозапуск/майнинг-пулы) + многошаговое обратимое удаление по [`security/miner-removal.md`](security/miner-removal.md). Тестируемое ядро плана `MinerRemovalPlanner` готово (с тестами); сигналы-входы готовы (CPU-сэмплинг процессов, имя процесса в подключениях). **Осталось — продюсер находки-майнера и исполнитель шагов** (`StopProcesses`/`DisableAutostart`/`QuarantineFiles` переиспользуют готовые правки; `ScheduleDeleteOnReboot` = `MoveFileEx`, `RequestReboot`). Эти разрушающие Windows-примитивы нельзя проверить на Linux — строить и валидировать на живой Win11 (смыкается с ребут-флоу Фазы 3), не «вслепую».
- [x] Базовая сверка с Windows Defender (MpCmdRun, локально/офлайн) — `DefenderScanner`, часть онлайн-проверки.
- [ ] Честно отразить ограничение: полноценный AV-движок с сигнатурами с нуля НЕ делаем.

## Фаза 5 — Драйверы _(в основном готово)_
- [x] Определение точной модели ПК/ноутбука (производитель + модель, WMI Win32_ComputerSystem) — `DriverProbe`/`DriversScanner`.
- [x] Поиск **недостающих**/проблемных драйверов (устройства без драйвера — тачпад, Wi-Fi, звук, видео) и установка по кнопке «Найти драйвер» через ОФИЦИАЛЬНЫЙ механизм Windows (`pnputil /scan-devices`). Автоскачивание со сторонних сайтов сознательно НЕ делаем (ненадёжно для произвольной модели) — честно отражено в UX.
- [ ] Проверка уже установленных драйверов на новые версии (онлайн-каталог — позже).
- [x] Звук: объяснение схемы аудио простыми словами (встроенный / HDMI-звук видеокарты) + поиск «улучшайзеров» (Nahimic/Dolby/MaxxAudio) с обратимым отключением службы — `AudioScanner`/`AudioProbe` (находки в вкладке «Драйверы»).
- [~] Видеокарта: определяется и показывается (`Win32_VideoController`) + рекомендация. Глубокий тюнинг NVIDIA/AMD (NVAPI/ADLX) — отдельно, best-effort.

## Дополнения (одобрено заказчиком 2026-06-17) _(ПЛАН)_

Сверх 10 исходных функций — согласованы с Иваном:

- [x] **Здоровье диска (SMART)** — шкала-индикатор 🟢/🟡/🔴 по параметрам диска + вывод «норма / стоит присмотреть / опасно» простыми словами. Ранний сигнал об умирающем диске. _(`DiskHealthScanner` + `DiskHealthProbe` (WMI MSFT_PhysicalDisk) готовы)_
- [x] **Угрозы — глубже:** проверка файла `hosts` и DNS + подозрительные сетевые подключения процессов. _(`NetworkThreatScanner` + `NetworkThreatProbe` (hosts/DNS/TCP) готовы)_
- [x] **«Доверять/игнорировать» находки** — кнопка «Безопасно» на находках с файлами (процессы/автозапуск): помеченный путь не всплывает при следующих сканах (`IWhitelist`/`WhitelistStore`, хранится локально).
- [~] **Температуры CPU/GPU** — показ температур со шкалой 🟢/🟡/🔴 и вердиктом простыми словами (`TemperatureScanner` + `TemperatureProbe`: CPU через ACPI, GPU через nvidia-smi). Глубокий тюнинг частот/троттлинга GPU (NVAPI) — отдельно, Фаза 5.
- [x] **Поиск больших и дублирующихся файлов + анализатор места** — освободить место сверх «мусора». _(`LargeDuplicateScanner` + `FileInventoryProbe`; плюс `DiskUsageScanner` + `DiskUsageProbe`: заполненность дисков 🟢/🟡/🔴 и крупнейшие папки профиля)_
- [x] **Удаление встроенного UWP-хлама** — Candy Crush, Bing News/Weather, пасьянсы, промо-приложения (белый список, нужное не трогаем); обратимо (вернуть из Microsoft Store). _(`AppxBloatScanner` + `AppxProbe` + `AppxRemoveFix` + `AppxRemovalBackupStore`)_
- [x] **Приватность/деблоат Windows** — найти и (по кнопке, обратимо) отключить: телеметрию (всегда со статусом), рекламный ID, рекламу в Пуске, персонализацию + службы Xbox. _(`PrivacyDebloatScanner` + `PrivacyProbe` готовы; правки реестра/служб обратимы)_
- [x] **Звук (по запросу Ивана)** — схема звука простыми словами + «улучшайзеры» (Nahimic/Dolby/MaxxAudio), которые портят микрофон/стрим: обратимое отключение службы. _(`AudioScanner` + `AudioProbe`, вкладка «Драйверы»)_
- [x] **Фирменные утилиты под модель ПК** — Lenovo Vantage / Armoury Crate / MSI Center / G HUB / Synapse / iCUE…, тихая установка через встроенный `winget`, где недоступен — открытие офиц. страницы. _(`UtilitiesScanner` + `UtilitiesProbe` + `WingetInstallFix`, вкладка «Утилиты»)_

> **Сквозной UX-принцип** (по требованию заказчика): аудитория — люди, которые **вообще не разбираются в ПК**. Каждая находка/параметр даёт явный вердикт простыми словами («нормально / стоит присмотреть / опасно»), без жаргона. Зафиксировано в [`coding-standards`](../.claude/rules/coding-standards.md).

## Быстрый старт (Фаза 1) _(выполнено)_

> Скелет уже создан этими шагами (решение `Aegis.sln`). Команды сборки/запуска — в [`BUILD.md`](BUILD.md).

1. Создать проект Avalonia на .NET 9:
   ```bash
   dotnet new wpf -n Aegis -f net9.0
   ```
2. Добавить базовые пакеты:
   ```bash
   dotnet add package CommunityToolkit.Mvvm   # MVVM (ObservableObject, RelayCommand)
   dotnet add package Serilog                 # логирование
   dotnet add package System.Management       # WMI (системная информация)
   ```
3. Подключить `app.manifest` с `requireAdministrator` (запуск с правами администратора).
4. Single-file self-contained publish под win-x64:
   ```bash
   dotnet publish -c Release -r win-x64 --self-contained true -p:PublishSingleFile=true
   ```
