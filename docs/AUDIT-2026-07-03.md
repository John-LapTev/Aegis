## 🔴 Критично — безопасность удалений (могут удалить лишнее)

1. **LeftoverService: удаление огромных общих папок вендора.** `CandidateNames` берёт первое слово имени программы → `%LOCALAPPDATA%\<word>` / `%APPDATA%\<word>` / `%PROGRAMDATA%\<word>`. Удаление «Microsoft Teams» предложит снести `AppData\Roaming\Microsoft` (Credentials, ключи Protect/Crypto, OneDrive, .NET), «Google Chrome» → `%LOCALAPPDATA%\Google` (все профили/пароли). Нет проверки `IsSafeToDelete`. `LeftoverService.cs:55-63,167-172`.
2. **LeftoverService: InstallLocation может быть `C:\` / `Program Files`.** `NormalizePath` не отсекает корневые пути → скан добавит их и предложит в Корзину. `LeftoverService.cs:55`.
3. **StartupProgramRemover: снос всего `AppData\Roaming`.** `IsSafeToDelete` блокирует только Windows и точный Program Files, но не профиль/AppData/ProgramData. Стартап-exe в `AppData\Roaming\updater.exe` → удалит всю папку Roaming. `StartupProgramRemover.cs:84-115`.
4. **Install-trace ловит чужие изменения.** Диф `after.Except(before)` по широким корням (Program Files/ProgramData/AppData 2 уровня) записывает любые папки/ветки реестра, созданные ДРУГИМИ процессами между снимками, как «добавленные этой программой» → потом удаляет их при «полном удалении». `ProgramUninstaller.cs:168-230`, `InstallSnapshotProbe.cs`.

## 🟠 Важно — корректность/ложные срабатывания

5. **MinerBehaviorScanner: пустое/короткое имя процесса раздувает вердикт майнера.** `entry.Command.Contains(process.Name)` при пустом Name матчит всё → лишний подтверждающий сигнал → Warning→Danger («очень похоже на майнер») ложно. `MinerBehaviorScanner.cs:178`. Аналогично мягче в GuardEvaluator.
6. **Раздел «Тесты»: GPU не нагружается, но его (простойная) температура подаётся как «под полной нагрузкой».** Тест грузит только CPU, но вердикт пишет «видеокарта до X°C под полной нагрузкой» и может дать Warning по простойному GPU. Вводит в заблуждение. `StressTestEngine.cs:46,181-209`.
7. **StartupProgramRemover: совпадение по префиксу без разделителя.** `C:\Program Files\Foo` матчит `C:\Program Files\FooBar\...`; `C:\` матчит всё → запуск чужого деинсталлятора. `StartupProgramRemover.cs:52`.
8. **ParseCommand: незакавыченный путь с пробелами** (`C:\Program Files\App\uninst.exe /S`) режется по первому пробелу → «Не удалось запустить удаление». `ProgramUninstaller.cs:106-120`.
9. **BootPerformanceScanner: предлагает отключить системные службы** (SysMain/wuauserv/WSearch/DiagTrack) — allow-list по exe-именам, не по service-name. `BootPerformanceScanner.cs:116-134`.
10. **Снимок сохраняется всегда, даже при деградации пробника** → следующий скан заливает «новыми» находками. `ChangesScanner.cs:42`, `AutostartChangesScanner.cs:42`.
