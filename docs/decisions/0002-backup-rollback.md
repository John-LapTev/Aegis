# 0002. Бэкап и откат изменений

- Статус: Accepted
- Дата: 2026-06-17

> ПЛАН. Кода ещё нет — фиксируем архитектурное решение до начала разработки.

## Контекст
Aegis вносит изменения в систему (реестр, автозапуск, службы, настройки). Любая правка
потенциально опасна: неудачное исправление может привести к глюкам или к тому, что ПК
**не загрузится**. Требования от заказчика (Ивана):

- **Бэкап ДО исправления** — всегда, перед любой правкой.
- Если после починки ПК не грузится/глючит — **откат при старте** (подхватить бэкап).
- Если починка требует перезагрузки — после ребута программа **сама запускается** с
  вопросом «всё ли работает корректно?»: «Да» → изменения остаются; «Нет, откатиться»
  → грузим бэкап.
- Отдельный **раздел «Бэкапы»**: выбрать точку/файл и восстановить вручную.
- Главный принцип: ничего не ломать безвозвратно, всё обратимо.

## Варианты
1. **Windows System Restore + экспорт затронутых веток реестра ПЕРЕД правкой**, откат
   при незагрузке — через **штатное восстановление Windows (WinRE / точка
   восстановления)**, авто-старт после ребута через **RunOnce/Task Scheduler** →
   диалог «всё ок? Да/Откатиться». Опирается на проверенные механизмы Windows, низкий
   риск «кирпича», понятный пользователю откат.
2. **Полный кастомный образ системы / самописный загрузчик (bootloader)** для отката
   при незагрузке — максимальный контроль, но **высокий риск «кирпича»**: ошибка в своём
   bootloader или процессе снятия образа может сделать систему незагружаемой; огромная
   сложность и поверхность для багов. **Отклонён.**
3. **Только экспорт веток реестра без System Restore** — легче, но не покрывает
   изменения вне реестра (службы, файлы, драйверы) и не даёт отката при незагрузке.
   Недостаточно надёжно. Отклонён как единственный механизм (берём как дополнение).

## Решение
Бэкап/откат строим на **двух уровнях**:

1. **Точка восстановления Windows (System Restore)** — создаётся **перед** пакетом
   изменений (сервис `RestorePointService`).
2. **Экспорт затрагиваемых веток реестра/настроек ПЕРЕД каждой правкой** — точечный
   обратимый бэкап конкретного `Fix` (каждое исправление создаёт точку восстановления/
   бэкап ветки ДО изменения).

**Откат при незагрузке** — только через **штатное восстановление Windows (WinRE /
точка восстановления)**, НЕ кастомный загрузчик. **Авто-запуск после ребута** —
через **RunOnce / Task Scheduler** → диалог «всё ок? Да / Откатиться». **Раздел
«Бэкапы»** в UI: список точек/экспортов, ручной выбор и восстановление.

## Последствия
- **Плюсы:** минимальный риск «кирпича» (опора на проверенные механизмы Windows);
  понятный пользователю откат; два уровня защиты (системная точка + точечный бэкап
  ветки); ручное восстановление из раздела «Бэкапы».
- **Минусы / риски:** System Restore должна быть **включена** в системе (защита диска) —
  на старте проверять и при необходимости предлагать включить; точки восстановления
  занимают место и не покрывают абсолютно всё (например, пользовательские файлы);
  RunOnce/Task Scheduler-флоу требует аккуратной обработки прерванного ребута.
- **Требует от остального стека:** `RestorePointService` (Фаза 1); каждый `Fix`
  обязан создавать обратимый бэкап ПЕРЕД правкой (см. [0004](0004-elevation-and-safety.md));
  флоу перезагрузки и раздел «Бэкапы» (Фаза 3); логирование всех бэкап/откат-операций
  (Serilog).
