# 0006. UI-фреймворк: Avalonia вместо WPF

- Статус: Accepted
- Дата: 2026-06-17
- Заменяет часть [0001](0001-stack.md) (выбор WPF для UI)

## Контекст
По [ADR 0001](0001-stack.md) UI планировался на **WPF**. Но появилось практическое требование
(от John, владельца среды разработки): **собирать готовый `.exe` на машине разработки (Linux) и
отправлять заказчику (Иван) один файл**. У WPF здесь жёсткое ограничение:

- **WPF собирается только на Windows** — XAML-компилятор WPF (MarkupCompilePass) и
  `Microsoft.NET.Sdk.WindowsDesktop` не работают на Linux. На Linux-машине разработки `.exe`
  с WPF-окном собрать **невозможно**.

Ядро (`Aegis.Core`, `Aegis.Scanners`, `Aegis.Threats`) от UI не зависит и уже собирается/тестируется
на Linux (75 тестов). Проблема только в слое окна.

## Варианты
1. **Avalonia (кроссплатформенный XAML UI)** — очень близок к WPF (XAML + MVVM, тёмная тема,
   стили), но собирается на любой ОС и **публикует самодостаточный Windows `.exe` прямо с Linux**
   (`dotnet publish -r win-x64 --self-contained`). Ядро/сканеры не затрагиваются; переписывается
   только слой окна.
2. **Остаться на WPF, собирать на Windows** (ПК заказчика / Windows-VM / CI) — UI богаче «из
   коробки», но `.exe` нельзя собрать на машине разработки; нарушает требование. Отклонён.
3. **WinUI 3 / MAUI** — те же проблемы со сборкой не на Windows и более тяжёлая упаковка.
   Отклонён.

## Решение
Берём **Avalonia 11** для `Aegis.App`. Проект таргетит **`net9.0-windows`** (без `UseWPF`), что:
- позволяет ссылаться на Windows-слой `Aegis.System` (`net9.0-windows`: WMI/реестр/P-Invoke);
- **собирается на Linux** (targeting-pack качается из NuGet) и публикуется в win-x64 `.exe`.

**Проверено 2026-06-17:** на Linux собрано `Aegis.exe` — самодостаточный single-file
PE32+ Windows GUI x86-64 (без установки .NET на машине пользователя). Тёмная тема и макеты
([`../design/mockups/`](../design/mockups/)) переносятся на Avalonia (тот же XAML-подход, токены те же).

## Последствия
- **Плюсы:** `.exe` собирается на машине разработки и отдаётся заказчику одним файлом; UI можно
  **компиляторно проверять на Linux** (раньше — только на Windows); кроссплатформенность на будущее.
- **Минусы / риски:** другой диалект XAML (стили через селекторы, шаблоны контролов иначе) — слой
  окна переписывается; Avalonia визуально чуть отличается от нативного WPF (для «для себя» приемлемо);
  single-file `.exe` крупный (~90+ МБ self-contained) — для отправки сжимать/或 framework-dependent.
- **Требует от остального стека:** `Aegis.App` — Avalonia + `net9.0-windows`; `app.manifest`
  (requireAdministrator) сохраняется; coding-standards и MVVM (CommunityToolkit.Mvvm) — без изменений;
  ViewModels/Services/SampleData переиспользуются как есть.
