# 0005. Драйверы и тюнинг GPU (политика применения)

- Статус: Accepted
- Дата: 2026-06-17

> ПЛАН. Кода ещё нет — фиксируем решение до начала разработки.

## Контекст
Группы функций 2 (драйверы — новые версии), 7 (тюнинг GPU NVIDIA/AMD) и 10
(недостающие драйверы/библиотеки) предполагают изменения, более рискованные, чем
правки реестра: установка драйверов и применение настроек видеокарты могут повлиять
на стабильность и загрузку системы. Нужно зафиксировать **что именно делает программа
сама**, а где оставляет решение пользователю.

Заказчик (Иван) ответил прямо: программа должна **и качать+ставить драйверы**, **и
показывать рекомендованные настройки видеокарты с объяснением «почему так лучше»**, **и
применять их по нажатию кнопки**. Целевая машина — **Windows 11, NVIDIA, Intel
i9-12900HX, 32 ГБ** (ноутбук); приоритет вендора GPU — **NVIDIA** (AMD-ветка
best-effort/позже).

## Варианты
1. **Полное авто-применение по кнопке + объяснение + обязательный бэкап/откат.**
   Программа находит новую версию драйвера и рекомендованные настройки, человеческим
   языком объясняет пользу, и по явному нажатию кнопки скачивает/ставит драйвер и
   применяет настройки — каждый шаг через обратимую операцию (точка восстановления +
   экспорт затрагиваемых настроек/ветки ПЕРЕД правкой). Совпадает с запросом заказчика.
2. **Только рекомендации без применения** — программа советует, ставит/жмёт человек.
   Безопаснее, но заказчик явно попросил применение по кнопке. Отклонён как единственный
   режим (оставляем как поведение по умолчанию для самых рискованных настроек —
   см. ниже).
3. **Тихое авто-применение без подтверждения** — недопустимо: нарушает принцип явных
   предупреждений и контроля пользователя ([0004](0004-elevation-and-safety.md)).
   Отклонён.

## Решение
**Вариант 1.** Для драйверов и настроек GPU действует единый паттерн:
**найти → объяснить простыми словами («что и почему лучше») → применить по явной кнопке
→ через обратимую операцию.**

- **Драйверы:** проверка наличия новой версии и установка по кнопке. Источник — только
  официальный (предпочтительно вендорский канал NVIDIA / каталог Windows Update), без
  сторонних «драйвер-паков». Откат — штатный механизм Windows (откат драйвера / точка
  восстановления). Установка драйвера, требующая ребута, идёт через
  [reboot-flow](0002-backup-rollback.md) («всё ок? Да / Откатиться»).
- **Настройки GPU (NVIDIA):** рекомендации формируются под мощность ПК; перед
  применением экспортируется текущее состояние (NVAPI/реестр вендора) для отката.
  Применение — по кнопке.
- **Градация риска:** безопасные настройки можно применять «в один клик»; настройки с
  риском нестабильности (агрессивные performance-режимы) — с отдельным явным
  предупреждением и подтверждением.
- **Ограничение honesty:** разгон/настройки, которые железо/драйвер не поддерживают,
  не навязываем; на ноутбучных GPU часть параметров недоступна — честно показываем это
  в UI.

## Последствия
- **Плюсы:** удобство «по кнопке» при сохранении принципа обратимости и понятных
  объяснений; пользователь всегда жмёт сам и видит, что произойдёт.
- **Минусы / риски:** установка драйверов и GPU-настройки — повышенный риск (стабильность,
  ребут); обязателен бэкап/откат и явные предупреждения; загрузка драйверов требует сети
  и надёжного определения официального источника и модели GPU; ноутбучные NVIDIA-GPU
  ограничивают часть настроек.
- **Требует от остального стека:** `DriversScanner`/`GpuScanner` как `IScanner`-плагины с
  `Finding`(Explain) (Фаза 5); `GpuTuner` в `Aegis.System/Gpu` (NVAPI/`nvidia-smi`,
  best-effort); каждое применение — обратимо ([0002](0002-backup-rollback.md),
  [0004](0004-elevation-and-safety.md)); Serilog на установку драйвера и применение
  настроек.
