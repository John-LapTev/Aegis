# Разбор чужих инструментов: Kudu и Sophia Script (2026-07-23)

Иван прислал два проекта с вопросом «что перенять для Aegis». Ниже — разбор кода и
конкретные выводы. Отдельный акцент на игровых фиксах (просьба Ивана).

| Проект | Что это | Стек | Лицензия |
|--------|---------|------|----------|
| [Kudu](https://github.com/adventdevinc/kudu) | Чистильщик + сканер безопасности (Windows/macOS/Linux) | Electron + TypeScript, ~24k строк IPC-логики | MIT |
| [Sophia Script](https://github.com/farag2/Sophia-Script-for-Windows) | Тюнер Windows 10/11 | PowerShell, модуль ~11.7k строк, ~150 функций | MIT |

Лицензии разрешают заимствование, но правило проекта прежнее: **clean-room** — переносим
идеи и подход, код пишем свой (как с GhostHunter/Win11Debloat).

---

## 1. Игры — чего у нас нет

### 1.1 «Игровой режим» (Kudu `game-mode.ipc.ts`) — главная находка
Временная, полностью обратимая оптимизация на время игры. Что делает при активации,
записывая исходное состояние в снимок на диск:

| Действие | Что сохраняется для отката |
|----------|---------------------------|
| Останавливает службы `WSearch`, `SysMain`, `wuauserv`, `Spooler`, `DiagTrack` | исходный тип запуска + «была ли запущена» |
| Закрывает фоновое: браузеры, мессенджеры (Discord/Slack/Teams/Telegram), апдейтеры (GoogleUpdate, AdobeARM, jusched…) | список PID+имя (информационно) |
| Схема питания → «Высокая производительность» | GUID прежней схемы |
| Фокусировка внимания (не беспокоить) | прежнее состояние |
| Запрет засыпания на время игры | id блокировщика |
| Отключает Game Bar / GameDVR (`AppCaptureEnabled`, `GameDVR_Enabled`) | прежние значения реестра |
| Отключает «оптимизацию во весь экран» (FSE behavior) | прежние значения |
| Отключает прозрачность интерфейса | прежнее значение |
| Отключает алгоритм Нейгла на сетевых адаптерах (`TcpNoDelay=1`, `TcpAckFrequency=1`) | прежние значения по каждому интерфейсу |

Ключевые детали реализации, которые стоит скопировать по смыслу:
- **Снимок на диске** переживает перезапуск приложения — режим можно выключить даже после
  падения программы. Это ровно наша модель обратимости, но для временных правок.
- **Снимок валидируется при чтении** (белые списки имён служб, путей и имён значений реестра,
  формат GUID, диапазоны чисел). Защита от подмены файла — нам обязательно, файл лежит в
  профиле пользователя.
- **Защищённый список процессов** (`csrss`, `lsass`, `services`, `winlogon`, `explorer`…) —
  никогда не убиваются, плюс отсечка `pid <= 4`.
- **Авто-детект игры**: опрос `tasklist` раз в 10 с по списку известных игровых exe
  (~70 тайтлов) + пользовательские; при выходе игры всё возвращается само. Есть подавление
  повторной активации, если пользователь вручную выключил режим при запущенной игре.

Для нас это одна большая фича «Игровой режим» на дашборде: одна кнопка, честный откат,
авто-режим по желанию.

### 1.2 Игровые твики Sophia
- `GPUScheduling` (аппаратное планирование GPU, `HwSchMode=2`) — **с гейтами**: только если
  есть дискретная видеокарта, ОС не в виртуалке и WDDM ≥ 2.7. Хороший пример «умного гейта».
- `XboxGameBar`, `XboxGameTips` — отключение Game Bar насовсем (у нас есть только как
  приватность в `PrivacyProbe`, отдельного игрового смысла не даём).
- `Install-VCRedist`, `Install-DotNetRuntimes` — установка Visual C++ Redistributable и
  .NET-рантаймов. Это самая частая причина «игра не запускается / вылетает при старте».
  У нас есть `WingetInstallFix` — фича собирается из готовых кубиков.
- `PowerPlan` — важная тонкость: перед сменой схемы Sophia **снимает политику**
  `Policies\Microsoft\Power\PowerSettings\ActivePowerScheme`, иначе изменение не применяется
  и не видно в интерфейсе.

### 1.3 Чистка игрового мусора (Kudu `rules/win32/gaming.json`, `gpu-cache.json`)
У нас в `AppCacheCatalog` покрыты: Steam, Epic, EA/Origin, GOG, Battle.net, шейдеры
NVIDIA/AMD/DirectX, Unity. **Не покрыто** (есть у Kudu):

- лаунчеры: Ubisoft Connect, Riot Client, Xbox App (UWP LocalCache), Rockstar Launcher,
  Amazon Games, itch.io, Overwolf/CurseForge;
- игры напрямую: Valorant, Fortnite, Roblox, Minecraft (logs + crash-reports + webcache);
- шейдеры: Intel `ShaderCache`, AMD `VkCache` (Vulkan), NVIDIA `NV_Cache` в `ProgramData`;
- Epic `VaultCache` и `Intermediate` — часто десятки гигабайт.

Дёшево (данные, не код) и сразу заметно по освобождённому месту.

## 2. Новые сканеры, которых у нас нет

Отсортировано по пользе для нашего пользователя.

1. **Оставшиеся политики Windows** (Sophia `ScanRegistryPolicies`). Сканирует
   `Policies`-ветки в HKLM/HKCU и сверяет найденное с ADMX-шаблонами из
   `%SystemRoot%\PolicyDefinitions`. Находит политики, оставленные чужими «оптимизаторами»
   и активаторами: отключённый Защитник, заблокированные обновления, запрет диспетчера задач.
   Для человека это выглядит как «Windows сломана и ничего не помогает» — а виновата
   забытая политика. Сильная и уникальная находка, у нас такого нет.
2. **Старые пакеты драйверов в DriverStore** (Kudu `driver-manager.ipc.ts`). `pnputil -e`
   → группировка по производителю+классу → самый новый считается текущим, остальные
   (и не привязанные к железу) — устаревшие. Освобождает единицы гигабайт. У нас `pnputil`
   используется только для установки/поиска.
3. **Аудит правил брандмауэра** (Kudu `firewall-audit.ipc.ts`). Ищет правила, разрешающие
   входящие соединения кому угодно, правила от удалённых программ (битый путь), правила не
   от Microsoft. Отличает системные правила по MUI/AppX-ссылке в описании и держит белый
   список (mDNS, LLMNR, Tailscale, Docker). У нас проверяется только «брандмауэр включён».
   **Здесь же наш баг:** `SettingsProbe` читает только `StandardProfile` (частная сеть) —
   если выключен публичный или доменный профиль, мы этого не увидим. Надо читать все три.
4. **Чистка контекстного меню проводника** (Kudu `context-menu-cleaner.ipc.ts`). 12 веток
   (`HKCR`/`HKCU` × `*`/`Directory`/`Background`/`Folder`/`Drive`/`AllFilesystemObjects`),
   поиск пунктов от удалённых программ. Отключение **обратимое**: у обработчиков имя ключа
   получает префикс `-`, у команд ставится `LegacyDisable` — ничего не удаляется. Лечит
   «правый клик думает 5 секунд».
5. **Чистка переменных среды** (Kudu `environment-cleaner.ipc.ts`). Записи `PATH`, ведущие
   в несуществующие папки, и «висячие» переменные вроде `JAVA_HOME` от удалённых программ.
   Защита: никогда не писать пустой `PATH`, перечитывать значение перед правкой, рассылать
   `WM_SETTINGCHANGE`.
6. **Оптимизация баз SQLite** (Kudu `database-optimizer.ipc.ts`) — `VACUUM` для баз браузеров
   и приложений. Ускоряет запуск браузера, экономит место.
7. **TRIM / оптимизация дисков** (Kudu `disk-trim.ipc.ts`) — `Optimize-Volume` (TRIM для SSD,
   дефрагментация для HDD) с историей запусков.
8. **Расписание** (Kudu `scheduler.ts`, Sophia `CleanupTask`/`TempTask`/
   `SoftwareDistributionTask`). Автопроверка раз в неделю/месяц с уведомлением. Sophia делает
   это задачами планировщика с тост-уведомлением — работает без запущенной программы.
9. **Массовое обновление программ** (Kudu `software-updater.ipc.ts`) — winget + Chocolatey +
   Scoop, одна кнопка «обновить всё». У нас winget используется точечно.

## 3. «Здоровье безопасности» — чего не хватает нашему разделу

Kudu собирает картину безопасности (`platform/win32/security.ts`), Sophia — умеет включать.
Нам добавить как находки с вердиктом простыми словами:

- шифрование диска BitLocker (`Get-BitLockerVolume`) — включено/нет;
- **все три** профиля брандмауэра, а не один;
- давность последнего обновления Windows (`Get-HotFix` → «последняя заплатка 94 дня назад»);
- блокировка экрана: заставка с паролем, `InactivityTimeoutSecs`;
- Windows Hello (PIN/лицо/отпечаток) и политика паролей (`net accounts`, `secedit`);
- слушающие порты (netstat LISTENING) — «что торчит наружу»;
- из Sophia: защита LSA (`RunAsPPL`, с проверкой аппаратной виртуализации), защита сети
  Defender (`Set-MpPreference -EnableNetworkProtection`), обнаружение потенциально нежелательных
  программ (`-PUAProtection`), песочница Defender, SmartScreen для приложений, DNS-over-HTTPS.

## 4. Инженерные приёмы, которые стоит перенять

1. **Снимать политику перед правкой.** Sophia почти в каждой функции сначала удаляет
   соответствующее значение из `...\Policies\...` и только потом пишет настройку. Иначе GPO
   перебивает пользовательское значение: мы отрапортуем «Исправлено», а система проигнорирует.
   Прямая угроза нашей честности — стоит добавить общую проверку «поверх настройки лежит
   политика» в `RegistryValueFix`.
2. **Гейты по железу и версии ОС перед твиком.** HAGS — только при дискретной GPU и WDDM ≥ 2.7;
   LSA PPL — только при аппаратной виртуализации; Sophia держит минимальные поддерживаемые
   номера сборок Windows в `supported_windows_builds.json` и отказывается работать на старых.
   У нас гейтов почти нет — на неподходящей машине твик бесполезен или вреден.
3. **Валидация состояния, прочитанного с диска** (Kudu `validateSnapshot`): белые списки
   путей/имён, диапазоны значений. У нас индексы бэкапов читаются доверчиво.
4. **Нативные утилиты вместо PowerShell в частых опросах.** Kudu специально ушёл с PowerShell
   на `netstat` для опроса соединений каждые 30 с: «~1 МБ вместо ~80 МБ и 2–5 с на запуск».
5. **Правила чистки — данными, а не кодом.** У Kudu чистильщики описаны JSON-файлами со
   схемой (`rules/win32/*.json`), новая программа добавляется без изменения кода. У нас
   `AppCacheCatalog` — это C#-код; данные вынести проще для роста.
6. **Обновляемая база правил сканера угроз.** Kudu грузит YARA-правила с сервера (проверка
   sha256, лимиты на размер и количество, обновление раз в 6 часов) и компилирует одним
   вызовом — «в 240 раз быстрее», чем по файлу. Нам подойдёт свой формат правил, обновляемый
   через GitHub Release (канал уже есть).
7. **Токены для поиска остатков** (Kudu `uninstall-leftovers.ts`): из списка установленных
   программ строятся токены — имя, первое слово, имя без версии, издатель, папка установки —
   и папка считается остатком, если не совпала ни с одним. Наш `ProgramLeftoverScanner` ищет
   только пустые папки; их подход находит и папки с данными. Стоит совместить с нашей
   `NameMatch.ReferencesWholeWord` (у них голое совпадение подстроки — тот самый Rave→Brave).

## 5. Что НЕ брать

- **«Очистка standby-памяти» у Kudu** — там вызывается сборщик мусора .NET в PowerShell
  (`[System.GC]::Collect()`), к системному списку ожидания это отношения не имеет. Фикция;
  честная реализация требует `NtSetSystemInformation`. Лучше не делать вовсе, чем делать вид.
- **CVE-сканер Kudu** — завязан на их облачный сервис и разбор версий пакетов Debian/RPM;
  для Windows-программ практически не применим.
- **CLI-режим, 30 языков, облачный агент** — не наш сценарий («для себя», интерфейс русский).
- **Массовое отключение служб** без разбора — у Kudu список служб в игровом режиме
  консервативный (5 штук), и это правильно; «оптимизаторы» с сотней отключённых служб как раз
  и создают проблемы, которые мы потом лечим.

## 6. Предлагаемый порядок

1. Игровой режим (обратимый, со снимком) + авто-детект игры.
2. Расширение каталога игровых кэшей (данные) + игровые твики с гейтами (HAGS, Game Bar,
   схема питания, VC++/.NET для игр).
3. Сканер оставшихся политик Windows.
4. Аудит брандмауэра + исправление проверки профилей (баг) + карточка «Здоровье безопасности».
5. DriverStore, контекстное меню, переменные среды.
6. Расписание проверок, TRIM, VACUUM баз, массовое обновление программ.

Пункты 1–3 дают самый заметный эффект; 4 включает починку существующего дефекта.
