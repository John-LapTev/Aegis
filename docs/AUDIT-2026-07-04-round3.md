Третий, финальный раунд 3-аудитного процесса Ивана (1277): (а) перепроверка корректности после всех правок раундов 1-2; (б) взгляд ВПЕРЁД — чего не хватает по логике/функционалу, что улучшить/расширить. Два независимых ревьюера.

## Часть А — корректность: ГОТОВО
Все правки раундов 1-2 подтверждены верными, ошибки не вернулись, новых регрессов нет. Сборка без warnings, 434 теста зелёные. Проверено детально: `swapStarted`-откат (все 3 ветвления сбоя — рабочий .exe возвращается всегда), SHA-256/размер/MZ, `ProcessStopFix` System32-проверка, кеш репутации, денилист дедупа во всех ветках, `ConnectivityWatcher` Dispose, `IsBusy`, сброс дедупа, статус поиска. Общий баг-свип удаления/остатков/сканов/автозапуска — новых багов нет.
Закрыты мелкие нюансы: `lsaiso` (Credential Guard) добавлен в критичные процессы; пустой/битый `.sha256` не блокирует обновление (уже проверены размер+MZ).
**Вывод: с точки зрения корректности программа готова.**

## Часть Б — чего не хватает (forward-looking, для приоритизации Иваном)

### 🔴 Главный разрыв «обещано vs сделано»: удаление майнеров не доведено
`docs/security/miner-removal.md` цитирует ключевое требование Ивана: не просто находить майнер, а **удалять** (или снять автозапуск + карантин + дочистить после ребута, всё обратимо). Реально: находка-майнер (`MinerBehaviorScanner`) даёт только действие `process-stop` → `ProcessStopFix` (kill одного процесса). Он НЕ снимает автозапуск, НЕ отправляет файл в карантин → майнер вернётся после перезагрузки. Многошаговый обратимый план `MinerRemovalPlanner` (Backup→StopTree→DisableAutostart→Quarantine/ScheduleDeleteOnReboot→Reboot) **написан, но осиротевший** (0 ссылок в продакшене, нет исполнителя `MinerRemovalFix`, нет примитива delete-on-reboot `MoveFileEx`). Роадмап честно помечает Фазу 4 как `[ ]`, но кнопка «Остановить» создаёт впечатление, что майнер убран. **Приоритет №1 forward-work** — это единственная невыполненная из 10 исходных функций, названная ключевой.

### 🟠 GPU-тюнинг (VISION #7) отсутствует целиком
VISION обещает «применяет рекомендованные настройки видеокарты по кнопке». В коде — 0 (`GpuTuner`/NVAPI/ADLX нет), только определение видеокарты + ссылка на драйвер. Либо реализовать безопасное обратимое подмножество, либо понизить статус в VISION до «позже».

### 🟠 «Драйверы — новые версии» только для NVIDIA
Реальная сверка «установлено vs последнее» есть только для NVIDIA GPU. Для AMD/Intel и прочих устройств — ссылка/поиск без сравнения версий. Либо дотянуть, либо явно сузить обещание.

### 🟡 Пробелы UX для аудитории «ноль в ПК» (высокая польза, низкая стоимость)
- **Нет онбординга/первого запуска** — приложение открывается сразу на «Сканах» без «что это и с чего начать».
- **Нет общего итога скана простыми словами** — после «Проверить всё» только «Проверка завершена» + находки по вкладкам; нет «светофора здоровья» и «нашли N важного, начни отсюда».
- **Нет кнопки «починить всё безопасное разом»** по итогам общего скана.
- **Нет отмены долгих операций** (скан/SFC/DISM/закачка обновления) — токен есть, не проброшен.
- Обратимость: приложение само не восстанавливает точку восстановления Windows (только пофиксовые бэкапы); стоит явно проговорить границы, чтобы «защищён» не читалось шире реального.

### Идеи улучшений (приоритет по пользе для аудитории)
1. **Довести удаление майнеров** (planner + исполнитель + delete-on-reboot) — закрывает ключевое обещание.
2. **Экран-итог скана + «светофор здоровья ПК» + «Исправить безопасное разом»** — снимает растерянность новичка (данные уже есть).
3. **Онбординг первого запуска** (2-3 шага: проверит → объяснит → перед правкой бэкап → ничего без спроса).
4. **Кнопка отмены долгих операций** (токен уже готов).
5. Драйверы AMD/Intel; GPU-тюнинг или понижение обещания; явные границы обратимости.

## Итог 3-аудитного процесса
Корректность и обратимость реестровых правок — крепкие и честные. Каркас чистый. Основное «осталось» — не баги, а **доведение обещанных фич** (майнеры — №1) и **UX для новичка** (итог скана, онбординг). Это продуктовые приоритеты — на решение Ивана.
