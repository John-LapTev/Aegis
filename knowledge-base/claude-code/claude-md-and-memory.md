# Claude Code: CLAUDE.md, правила и память

> Источник: code.claude.com/docs/en/memory · сверено 2026-05-30.

## Два механизма памяти (грузятся в начале каждой сессии)
- **CLAUDE.md** — инструкции, которые пишешь ты (человек). Правила, стандарты, архитектура.
- **Авто-память** — заметки, которые Claude пишет себе сам (находки, предпочтения). В `~/.claude/projects/<project>/memory/` (machine-local, общая для всех worktree одного git-репо).

Оба — это контекст, а не жёсткая конфигурация. Чтобы что-то ГАРАНТИРОВАННО блокировать/выполнять — используй **хуки**, а не CLAUDE.md.

## Где живёт CLAUDE.md (порядок загрузки: от общего к частному)
1. Managed policy (организация) — `/etc/claude-code/CLAUDE.md` (Linux).
2. Пользовательский — `~/.claude/CLAUDE.md` (все проекты).
3. Проектный — `./CLAUDE.md` или `./.claude/CLAUDE.md` (в git, для команды).
4. Локальный — `./CLAUDE.local.md` (личный, в `.gitignore`).

Файлы вверх по дереву каталогов загружаются целиком при старте; в подкаталогах — по требованию (когда Claude читает файлы там).

## `.claude/rules/` — официальный способ дробить правила
- Markdown-файлы в `.claude/rules/*.md` (рекурсивно). Одна тема — один файл.
- **Без `paths`-фронтматтера → грузятся при старте** с тем же приоритетом, что `.claude/CLAUDE.md`. Поэтому НЕ дублируй их ещё и через `@import` в CLAUDE.md (будет двойная загрузка) — достаточно ссылок.
- С `paths`-фронтматтером → грузятся только при работе с подходящими файлами (path-scoped):
  ```markdown
  ---
  paths:
    - "src/api/**/*.ts"
  ---
  # API rules ...
  ```
- Поддерживают симлинки (общие правила между проектами).

## Импорты в CLAUDE.md
- Синтаксис `@path/to/file` — относительный (от файла) или абсолютный, `~` для дома. Глубина рекурсии — до 4 переходов. Импортированные файлы грузятся при старте (не экономят контекст, только организуют).

## Лучшие практики
- Держи CLAUDE.md **< 200 строк**: длинные файлы хуже соблюдаются и едят контекст.
- Конкретика вместо общих слов («2-space indent», а не «format properly»).
- Убирай конфликтующие/устаревшие инструкции.
- Объёмные процедуры → в **скиллы** (грузятся по требованию), не в CLAUDE.md.
- `/init` генерирует стартовый CLAUDE.md; при существующем — предлагает улучшения.

## Авто-память
- `MEMORY.md` — индекс, грузится в каждую сессию (первые 200 строк / 25KB). Детальные заметки — в отдельных тематических файлах (грузятся по требованию).
- Управление — команда `/memory`. Можно отключить (`autoMemoryEnabled: false`).
