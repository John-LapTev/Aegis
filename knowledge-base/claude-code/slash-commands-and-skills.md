# Claude Code: слэш-команды и скиллы

> Источник: code.claude.com/docs/en/slash-commands, /skills · сверено 2026-05-30.

**Команды слиты со скиллами.** `.claude/commands/deploy.md` и `.claude/skills/deploy/SKILL.md` оба дают `/deploy`. Старые `.claude/commands/` работают; скиллы добавляют папку для вспомогательных файлов и авто-подгрузку по релевантности.

## Слэш-команда: `.claude/commands/<name>.md`
```markdown
---
description: Что делает команда
argument-hint: [аргументы]
allowed-tools: Read, Edit, Bash(git diff:*)
model: sonnet                 # опционально
---

Тело промпта. Подстановки:
- $ARGUMENTS — все аргументы; $1, $2 — позиционные.
- !`команда` — выполнить bash и подставить вывод (нужно разрешение в allowed-tools).
- @path/to/file — вставить содержимое файла.
```
- Подкаталоги = неймспейсы: `.claude/commands/git/commit.md` → `/git:commit`.
- `disable-model-invocation: true` — команда доступна только человеку, Claude её не вызывает сам.

## Скилл: `.claude/skills/<name>/SKILL.md`
- Для процедур/чек-листов/многошаговых воркфлоу. Тело грузится **только при использовании** — длинные справочные материалы почти ничего не стоят, пока не нужны.
- Следует открытому стандарту Agent Skills; можно запускать в субагенте, инжектить динамический контекст.

## Когда что
- **CLAUDE.md / rules** — факты и правила, нужные всегда.
- **Скилл** — процедура/воркфлоу, нужная иногда (загрузка по требованию).
- **Хук** — то, что обязано срабатывать в конкретный момент жизненного цикла.
