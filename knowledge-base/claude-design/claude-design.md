## Что это
**Claude Design** — визуальная рабочая среда от Anthropic Labs: ты описываешь, что нужно, и Claude рисует готовый дизайн на холсте. Запущена 17 апреля 2026 (research preview), работает на **Claude Opus 4.7**. Доступна на планах Claude **Pro / Max / Team / Enterprise** по адресу **claude.ai/design** (иконка-палитра в левом меню).

Делает дизайн заметно лучше, чем Claude Code «с нуля», потому что это отдельный инструмент, заточенный под визуал: дизайны, интерактивные прототипы, слайды, one-pager'ы, лендинги, макеты мобильных приложений, дашборды. Выдаёт цельный самодостаточный артефакт (например, слайды в HTML, анимированный сайт на Three.js, дашборд с живыми графиками).

## Как работает
- Две зоны: **чат слева, холст справа**. Описываешь → Claude генерит первую версию → дорабатываешь через диалог, inline-комментарии, прямое редактирование и слайдеры (которые Claude делает сам под макет).
- **Входные данные:** текстовый промпт; загрузка картинок и документов (DOCX/PPTX/XLSX); можно навести Claude на твою кодовую базу; web-capture — захват элементов прямо с твоего сайта, чтобы прототип выглядел как реальный продукт.
- **Дизайн-система:** при онбординге Claude читает твою кодовую базу и дизайн-файлы и строит дизайн-систему команды — дальше каждый проект автоматически использует твои цвета, типографику, компоненты.
- **Экспорт:** внутренний URL организации, папка, Canva, PDF, PPTX, отдельные HTML-файлы.

## 🔗 Связка с Claude Code (главное для нас)
Самая важная фича: **handoff в Claude Code**. Когда дизайн готов, Claude Design упаковывает всё в «handoff bundle», написанный специально под Claude Code (производитель и потребитель — модели одной лаборатории, формат оптимизирован между ними, а не «компромиссный» как design-tokens). Этот bundle передаётся в Claude Code одной инструкцией — и Claude Code строит по нему реальный код.

## Когда применять (в Project-Forge)
- Проект с серьёзным UI/визуалом (лендинг, веб-приложение, продукт «на красоте», презентации, дашборды) → сначала сделать дизайн в **Claude Design**, затем handoff в Claude Code для реализации.
- Альтернатива/дополнение прямо в коде — плагин **Frontend Design** (см. [`frontend-design-plugin.md`](frontend-design-plugin.md)).
- Не нужно для CLI, ботов без UI, библиотек.

## Ограничения
- Research preview, только платные планы Claude. Лучшие результаты — когда есть кодовая база/дизайн-файлы для построения дизайн-системы.
