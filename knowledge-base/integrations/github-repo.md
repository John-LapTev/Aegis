# Интеграция: GitHub-репозиторий для проекта

Когда локальный каркас готов и сделан `git init`, **предложи** опубликовать проект на GitHub. Просто и по шагам.

## 1. Проверь, к какому аккаунту есть доступ
- `gh auth status` — на этой машине `gh` установлен и залогинен как **`John-LapTev`** (scopes: `repo, workflow, gist, read:org`). По умолчанию используем его; если появятся другие аккаунты — спроси, в какой.
- Запасной путь (если gh недоступен): **GitHub MCP** (`mcp__github__*`) или токен `GITHUB_PERSONAL_ACCESS_TOKEN` из `.mcp.json` (см. глобальный `~/CLAUDE.md`).

## 2. Спроси параметры
- Имя репозитория (по умолчанию = имя проекта).
- Приватный (по умолчанию **ДА**) или публичный.

## 3. Создай репозиторий
- `gh repo create John-LapTev/<name> --private --source=. --remote=origin`
- Или GitHub MCP `mcp__github__create_repository` → `git remote add origin <url>`.

## 4. Залей код — ТОЛЬКО с разрешения
- По `git-workflow`: commit/push только по явной команде. Покажи `git status`, спроси, потом `git push -u origin main`.

## Изоляция
- Remote ставится ТОЛЬКО на этот проект. Никогда не переиспользуй чужой remote. Проверяй `git remote -v`.

## Что показать пользователю
> «Создам **приватный** репозиторий `<name>` в `John-LapTev` и подключу к проекту. Запушу по твоей команде. Ок?»
