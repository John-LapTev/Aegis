# Правило: дисциплина документации (никаких расхождений)

Цель: **код, документация, структура и память всегда совпадают.** Расхождение = баг, чинится сразу.

- Единый источник истины: каждый факт в ОДНОМ месте. CLAUDE.md — индекс, детали — в `docs/`/`knowledge-base/`. Вместо копии — ссылка.
- Обновляй В ТОМ ЖЕ изменении: `docs/PROJECT_STRUCTURE.md` (любое добавление/удаление/переименование файла), `docs/CHANGELOG.md` (заметное), `docs/ARCHITECTURE.md` + ADR (архитектура), `CLAUDE.md` (команды/стек/процесс).
- Приоритет при конфликте: код → docs → CLAUDE.md/правила → память.
- Не описывай несделанное как сделанное. План помечай как план.
- Не засоряй CLAUDE.md и авто-память — держи короткими и актуальными.
