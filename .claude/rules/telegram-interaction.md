# Правило: общение через Telegram (Channels)

Проект можно вести из терминала И из Telegram (бот через Channels). Всегда учитывай, ОТКУДА пришло сообщение и КТО пишет.

## Отвечай в том же канале
- Сообщение пришло из **Telegram** → отвечай в **Telegram**. Канал ответа = канал входящего сообщения; не «уводи» ответ только в терминал.
- Пришло из терминала → отвечай в терминале.

## Кто пишет — определяй по id
- В allowlist может быть несколько человек. Определяй отправителя по его Telegram `chat_id` из входящего `<channel>`-блока сообщения.
- Сопоставление `id → человек` — в `.personal/accounts.md` (личное, не в репозитории). Обращайся к человеку по имени и веди ОТДЕЛЬНЫЙ контекст диалога с каждым — не путай собеседников.

## В Telegram нет интерактивного UI
- Интерактивные всплывающие вопросы с выбором (инструмент `AskUserQuestion`, пикеры/кнопки) в Telegram **не отображаются** — там только обычный текст.
- Когда диалог идёт через Telegram — **НЕ вызывай `AskUserQuestion`**. Задавай вопросы обычным текстом с **пронумерованными вариантами** (1 / 2 / 3) и проси ответить номером или словом.
- Сообщения держи короче и структурированнее (мобильный формат): без длинных таблиц, чёткие списки, по делу.

## В терминале — как обычно
- В обычной консоли интерактивные вопросы (`AskUserQuestion`) работают корректно — используй их там.
