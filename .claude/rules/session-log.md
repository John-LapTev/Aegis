# Правило: память диалога через `.session-log.md`

У Telegram-**bot** сессий Claude Code НЕ сохраняет надёжно основной транскрипт чата → `--resume`/история диалога не работают. Поэтому непрерывность ведём в файле проекта, а не в чате.

**Файл:** `.session-log.md` в корне проекта (append-only, gitignored).

**При старте сессии (перед первым ответом):** ПРОЧИТАЙ последние записи `.session-log.md` — это твоя память о прошлых разговорах и решениях. (SessionStart-hook также показывает хвост лога в контексте.)

**После каждого ЗНАЧИМОГО обмена** допиши в конец `.session-log.md` краткую запись:

```
## YYYY-MM-DD HH:MM — кто (ivan/john)
- Запрос: <в двух словах>
- Сделано/решено: <ключевое; какие файлы/функции/доки тронуты>
- Далее/открыто: <если есть>
```

Дисциплина:
- Кратко (3–6 строк на запись), по-русски. Только значимое — не логируй каждое «ок»/«спасибо».
- НИКОГДА не пиши в лог секреты/токены/ключи.
- Дату/время бери из системы; не выдумывай.
- Это и есть непрерывность между запусками — НЕ полагайся на чат-историю Claude Code.
