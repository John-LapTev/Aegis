# Правило: стандарты кода

> Стек: **C# 13 / .NET 9 + Avalonia** (десктоп под Windows 10/11; UI собирается в `.exe` на любой ОС, ADR 0006). Статус — в разработке; стандарты применяются с первой строки.

## Язык и сборка
- C# 13 / .NET 9, `nullable enable`, `<TreatWarningsAsErrors>` — собирать **без warnings**.
- Читай файл перед изменением. Имя = документация (без аббревиатур).

## Архитектура
- **MVVM** (CommunityToolkit.Mvvm): View ↔ ViewModel ↔ Model; в code-behind — только UI-специфика, бизнес-логика в VM/сервисах.
- **DI** — сервисы через контейнер, не статические синглтоны; зависимости — через конструктор.
- **async/await** для любых системных/сетевых операций — **не блокировать UI-поток**; никаких `.Result`/`.Wait()`.

## Доступ к системе (критично)
- P/Invoke, WMI (System.Management), реестр, службы — только в **изолированных безопасных обёртках** (отдельные сервисы), не разбросано по VM/View.
- Вся работа с реестром/службами — **параметризованно** и с обработкой ошибок; никаких хардкод-путей вперемешку с логикой.
- **Любое системное изменение — через обратимую операцию:** точка восстановления Windows + экспорт затрагиваемой ветки реестра/настройки **ПЕРЕД** правкой.
- **Никаких безвозвратных удалений** без бэкапа; явные предупреждения пользователю перед изменением.
- `app.manifest` → requireAdministrator; на входе — **проверка фактических прав администратора**, понятная ошибка если их нет.

## Ошибки и логирование
- Каждый внешний вызов (WMI/реестр/процессы/сеть/файлы) — в обработке ошибок; ошибка доходит до пользователя понятным сообщением **на русском**.
- В `catch` — `Exception ex` обрабатывать осмысленно, не глотать; не ловить `Exception` там, где нужен конкретный тип.
- **Serilog** — логировать критичные операции (каждое исправление, создание точки восстановления, откат, обращения к VirusTotal).

## UX-тексты (критично — аудитория не разбирается в ПК)
- Пользователь **вообще не разбирается** в компьютерах и характеристиках. Любой текст должен быть понятен такому человеку — никакого жаргона.
- Каждая находка/параметр даёт **явный вердикт простыми словами**: «это нормально» / «стоит присмотреть» / «опасно, нужно действовать». Человек должен сразу понять: волноваться или нет, безопасно или нет.
- «?»-Explain — простыми словами: в чём суть, чем грозит (или «не грозит»), что даст исправление и зачем.
- Для измеримых параметров (здоровье диска, температуры) — шкала-индикатор 🟢/🟡/🔴 (severity Ok/Warning/Danger) + короткий вывод по каждому пункту.

## Платформа Windows: коллизии и тестируемость (наработано практикой)
- **Коллизия namespace:** в коде под `Aegis.*` префикс `System.X` может резолвиться в наш `Aegis.System` (ошибка сборки `CS0234`). Пиши через `using System…;` и голый тип **или** `global::System.X` (например `global::System.IDisposable`). Не начинай тип с `System.` без директивы using.
- **Системные операции — за абстракцией, best-effort:** реальный доступ к WUA/реестру/WMI/процессам изолируй в отдельном классе Aegis.System за интерфейсом (пример: `IDriverUpdateCatalog`→`WindowsUpdateDriverCatalog`, `ILeftoverService`→`LeftoverService`). Guard `OperatingSystem.IsWindows()`, всё в try/catch → вне Windows/при сбое возвращай пусто/неуспех, НЕ роняй скан/UI.
- **Разработка на Linux:** системные операции вживую тут не воспроизводятся. Правило: **логику покрывай тестами через фейки** (матчеры, сборка находок, разбор данных — чистые функции), а сам системный вызов — Windows-only, его проверяет Иван на Win11. Так каждая фича имеет тестируемое ядро + отдельный тонкий системный слой.
- **Позднее связывание COM (WUA):** допустимо `dynamic` в одном изолированном best-effort пробнике (типизированного interop под win-only нет). Обкладывай каждое обращение try/get-хелпером.

## Добавление фичи (единый путь «находка → починка»)
- Сканер (`IScanner`) выдаёт `Finding` с `Data[kind]` из **`FindingKinds`** и ключами из **`FindingDataKeys`** (не магические строки). `FixFactory` по `kind` строит `IFix`. Обратимость — через бэкап-хранилища (реестр/карантин) ПЕРЕД правкой; `FixOutcome.Ok(backupId…)` только при реальном бэкапе, иначе `OkWithoutBackup`.
- Новая правка = новый `FindingKinds.*` + ветка в `FixFactory` + класс `IFix` + тест на маршрутизацию/поведение.

## Тесты
- **xUnit**. Новая логика → тесты; баг-фикс → тест, воспроизводящий баг.
- Покрывать прежде всего **логику сканеров и бэкапа/отката** (системные обёртки — за абстракциями, мокаются).
- Сборка **без warnings** (`TreatWarningsAsErrors`); компилируемые привязки Avalonia (`AvaloniaUseCompiledBindingsByDefault`) ловят отвязку XAML↔VM на сборке — не отключай.

## Безопасность
- Никаких секретов в коде. **VirusTotal API key** и **токен бота** — через окружение / `.personal/`, не в репозиторий.
- Валидация ввода на каждой границе (формы, данные сканов, входящие сообщения).
- Чистый код по [`project-hygiene`](project-hygiene.md): один файл — одна ответственность, без мёртвого кода, дублируется 2+ раз → в общее.
