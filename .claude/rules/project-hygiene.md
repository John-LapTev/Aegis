# Правило: чистота (без мусора)

- В репозитории — только нужное. Временное, дампы, логи, артефакты → в `.gitignore` или `.personal/`.
- Секреты, `.env`, `*.db`, `node_modules/` — никогда в репозиторий.
- Заменяешь — удаляй старое. Без `old`/`new`/`_copy`/`_backup` (история в git).
- Чистый код: один файл — одна ответственность; без мёртвого кода; константы именованы; дублируется 2+ раз → выноси в общее.
- Размеры (ориентир): компонент ~400/500, хук ~300/400, сервис ~400/500 строк (soft/hard).
- Именование: `PascalCase` компоненты, `useCamelCase` хуки, `camelCase` утилиты, `UPPER_SNAKE_CASE` константы, `handle*`/`on*`/`is*`. Без аббревиатур.
